# msk_with_iam_message_sender_json

This project contains source code and supporting files for an application that you can run on an EC2 machine in the same account as where you have deployed the lambda function that is configured as part of the project msk_with_iam_consumer_dynamo_sam. It includes the following files and folders.

- msk_with_iam_message_sender_json/src/main/java - Code for the application that will send JSON payload messages to MSK to be consumed by the lambda function that is configured as part of the project msk_with_iam_consumer_dynamo_sam.

- msk_with_iam_message_sender_json/src/main/resources/us-500.csv - This is a file that has data for 500 fictitious persons. Each row in this CSV will be converted into a JSON payload and will be sent out as the payload of a Kafka message that will be generated by utilizing the Java code that is in msk_with_iam_message_sender_json/src/main/java, as explained in subsequent steps.

- pom.xml - A Maven POM file that will be used to build the Java project to generate an UberJar file that can be used to run the application that will send JSON payload messages to MSK

## Create MSK Cluster in AWS Account

Use a Cloudformation template to create an MSK cluster or create an MSK using the AWS console, in your AWS account. You can look at using the Cloudformation template specified under https://github.com/aws-samples/serverless-patterns/tree/main/msk-cfn-sasl-lambda/create-cluster-cfn. The Java program under msk_with_iam_message_sender_json will be sending Kafka messages to this MSK cluster. The Lambda function that will be created per instructions under msk_with_iam_consumer_dynamo_sam/README.md will consume the Kafka messages from the same MSK cluster

## Install Java, Maven

Before proceeding with the next step, please make sure you have Java JDK and Maven installed on your EC2 or local machine from where you will be running this program to send out Kafka messages to the MSK cluster

For the latest version of Amazon Corretto JDK (at the time of publishing), please go the following link:

https://docs.aws.amazon.com/corretto/latest/corretto-21-ug/downloads-list.html

Please follow the instructions to download and install the JDK for your Operating System

Note that you don't have to use Amazon Corretto JDK but can use JDK from another source as well

For the latest version of Maven (at the time of publishing) please go the following link:

https://maven.apache.org/download.cgi#:~:text=Apache%20Maven%203.9.9%20is,recommended%20version%20for%20all%20users.

Please follow the instructions to download and install Maven and then add the location to the bin folder of Maven in your System PATH

To ensure Java and Maven are correctly installed, run the commands:

java --version

mvn --version


## Install Apache Kafka and create topics

Once you have confirmed that Java and Maven are installed on the machine, the next step is to install Apache Kafka

You can download the open-source Apache Kafka from https://kafka.apache.org/downloads

It is a good idea to install the same version of Apache Kafka as the version of Kafka that you have configured in your MSK cluster

Choose Binary downloads. The version of Scala does not matter. Download the binary and then unzip the tar file to get the Apache Kafka installed

Once Kafka has been installed on the machine from where you will be sending Kafka messages to the MSK cluster, create a client.properties file in the local Kafka bin directory in the machine and add the following properties and save the file

security.protocol=SASL_SSL
sasl.mechanism=AWS_MSK_IAM
sasl.jaas.config=software.amazon.msk.auth.iam.IAMLoginModule required;
sasl.client.callback.handler.class=software.amazon.msk.auth.iam.IAMClientCallbackHandler

Also take a look at the https://github.com/aws/aws-msk-iam-auth/releases page for the latest version of the aws-msk-iam-auth-<version>-all.jar. Download this jar file and copy it into the libs folder of your Kafka installation. For example, you can use wget https://github.com/aws/aws-msk-iam-auth/releases/download/v2.2.0/aws-msk-iam-auth-2.2.0-all.jar


Next, we will be creating a Kafka topic. Use the following command format to create a Kafka topic

<Path to Kafka Installation Root Folder>/<Kafka Version>/bin/kafka-topics.sh --create --bootstrap-server <List of MSK Servers>:9098 --command-config client.properties --replication-factor 3 --partitions 3 --topic <Name of topic that you will use to send Kafka messages to MSK. The Lambda function should listen on the same topic>

For example, the above command could translate into something like this after the variables are substituted. Please make sure to substitute the right values of the variables for your own environment:

/home/ec2-user/environment/kafka_2.12-3.4.0/bin/kafka-topics.sh --create --bootstrap-server b-1.mskworkshopcluster.c50sxk.c13.kafka.us-west-2.amazonaws.com:9098,b-2.mskworkshopcluster.c50sxk.c13.kafka.us-west-2.amazonaws.com:9098,b-3.mskworkshopcluster.c50sxk.c13.kafka.us-west-2.amazonaws.com:9098 --command-config client.properties --replication-factor 3 --partitions 3 --topic MskIamJsonTestTopic

If you run into errors with the above command, please check the following:

- The security group of your MSK cluster allows inbound connections on the ports it needs from the security group of your EC2 machine or the IP address of your own machine if that's what you are using. For information on what ports MSK needs, take a look here - https://docs.aws.amazon.com/msk/latest/developerguide/port-info.html

- The IAM role of the EC2 machine has permissions necessary to access the MSK cluster. For testing purposes, you can assign the policy AmazonMSKFullAccess to the role, but in production, you should follow the least privilege policy and grant only permissions necessary to perform the job.


## Compile the application

After checking out from Git, cd to the folder Lambda-Event-Sources/Java/javaMSKLambdaDynamoDB/JSONMessage/msk_with_iam_message_sender_json relative from the root folder where you checked out from Git

You should see a pom.xml file in this follder

run  the command mvn clean install

This should build the code and create a jar file target/json-msk-iam-producer-0.0.1-SNAPSHOT.jar relative to the folder from where you ran the mvn clean install command


## Create Kafka Properties File

Create a properties file called kafkaProperties and note down the full path of where this property file is located

In the properties file, add the following properties

bootstrap.servers <List of MSK Brokers> - 

for example, 
bootstrap.servers=b-1.mskworkshopcluster.c50sxk.c13.kafka.us-west-2.amazonaws.com:9098,b-2.mskworkshopcluster.c50sxk.c13.kafka.us-west-2.amazonaws.com:9098,b-3.mskworkshopcluster.c50sxk.c13.kafka.us-west-2.amazonaws.com:9098

security.protocol=SASL_SSL
sasl.mechanism=AWS_MSK_IAM
sasl.jaas.config=software.amazon.msk.auth.iam.IAMLoginModule required
sasl.client.callback.handler.class=software.amazon.msk.auth.iam.IAMClientCallbackHandler

Save the properties file after adding these properties


## Send Kafka messages to MSK

Use the following command to send out Kafka messages to MSK. That should in turn trigger your lambda function that will consume these messages from MSK, parse the messages and write the contents of the messages to a DynamoDB table

java -classpath <root folder where project is checked out from Git>/Java/javaMSKLambdaDynamoDB/JSONMessage/msk_with_iam_message_sender_json/target/json-msk-iam-producer-0.0.1-SNAPSHOT.jar msk.iam.producer.JsonKafkaProducer <path to kafkaProperties file created in above step>/kafkaProperties <Topic on which to publish Kafka messages to MSK> <A unique key to identify this batch of messages> <Total number of messages to send - should be less than 500>

For example, the above command could translate into something like this after the variables are substituted. Please make sure to substitute the right values of the variables for your own environment:

java -classpath /home/ec2-user/environment/Lambda-Event-Sources/Java/javaMSKLambdaDynamoDB/JSONMessage/msk_with_iam_message_sender_json/target/json-msk-iam-producer-0.0.1-SNAPSHOT.jar msk.iam.producer.JsonKafkaProducer /home/ec2-user/environment/kafkaProperties MskIamJsonTestTopic TestKey01 100
